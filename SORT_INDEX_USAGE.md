# 排序基数功能说明

## 功能概述

在网盘自动转存系统中，当使用 `{I}`、`{II}`、`{III}` 等排序序号变量进行文件命名时，可以通过 `sort_index` 参数指定排序的起始数字。

## 使用场景

### 场景 1：续接已有文件编号

假设你的目录中已经有 5 个文件：
```
剧集 01.mp4
剧集 02.mp4
剧集 03.mp4
剧集 04.mp4
剧集 05.mp4
```

现在要转存新的剧集，希望从 6 开始编号，可以在任务配置中添加 `sort_index: 6`。

### 场景 2：特定编号规则

某些剧集或综艺可能从特定的数字开始编号，比如特别篇从 100 开始，可以在任务配置中设置 `sort_index: 100`。

## 配置方法

### 方法 1：通过 Web UI 配置

1. 打开 Web UI 管理界面
2. 在任务列表中找到对应的任务，或点击"增加任务"
3. 展开任务详情，找到"保存规则"部分
4. 在"排序基数"输入框中输入起始数字（如：5）
5. 保存配置

### 方法 2：直接在配置文件中编辑

在 `quark_config.json` 的任务配置中添加 `sort_index` 字段：

```json
{
  "tasklist": [
    {
      "taskname": "追剧任务",
      "shareurl": "https://pan.quark.cn/s/xxxxx",
      "savepath": "/剧集/某电视剧",
      "pattern": ".*\\.(mp4|mkv)",
      "replace": "{TASKNAME}.{SXX}E{II}.{EXT}",
      "sort_index": 5,  // ← 添加这一行，表示从 5 开始排序
      "enddate": "2099-12-31"
    }
  ]
}
```

## 使用说明

### 排序序号变量格式

- `{I}`：一位数序号（0-9）
- `{II}`：两位数序号（00-99）
- `{III}`：三位数序号（000-999）
- 以此类推...

### 示例

#### 示例 1：基本使用

配置：
```json
{
  "pattern": ".*?第 (\\d+) 期.*",
  "replace": "综艺名称.E{II}.第\\1 期.{EXT}",
  "sort_index": 10
}
```

分享文件列表：
```
综艺 第 1 期.mp4
综艺 第 2 期.mp4
综艺 第 3 期.mp4
```

转存后文件名：
```
综艺名称.E10.第 1 期.mp4
综艺名称.E11.第 2 期.mp4
综艺名称.E12.第 3 期.mp4
```

#### 示例 2：结合现有文件续接

目录中已有文件：
```
电视剧.S01E01.mp4
电视剧.S01E02.mp4
电视剧.S01E03.mp4
```

新任务配置：
```json
{
  "pattern": ".*\\.(mp4|mkv)",
  "replace": "电视剧.S01E{II}.{EXT}",
  "sort_index": 4  // 从第 4 集开始
}
```

转存后新增文件：
```
电视剧.S01E04.mp4
电视剧.S01E05.mp4
电视剧.S01E06.mp4
```

## 注意事项

1. **默认值**：如果不配置 `sort_index`，系统默认从 1 开始排序
2. **最小值**：`sort_index` 的最小值为 1
3. **与现有文件的关系**：如果目录中已有匹配的文件，系统会自动续接最后一个文件的编号
4. **序号位数**：确保 `{I}` 的数量足够，避免出现编号溢出的情况（如用 `{I}` 但编号超过 9）

## 兼容性

- 该功能完全向后兼容，不影响现有任务配置
- 旧配置不加 `sort_index` 参数，行为保持不变
- 仅在替换表达式中包含 `{I+}` 模式时生效

## 技术实现

系统在以下两个关键方法中添加了 `start_index` 参数：

1. `MagicRename.set_dir_file_list()`：初始化目录文件列表时考虑起始值
2. `MagicRename.sort_file_list()`：排序文件列表时使用起始值

通过在任务对象中添加 `sort_index` 字段，系统会在处理文件命名时自动应用该起始值。
